Nexus Game Intelligence System - Final Project Report

Abstract
This project focuses on the development of a centralized Database Management System (DBMS) named Nexus, designed to track and analyze the complex lifecycle of modern video games. The system integrates data from developers, publishers, and various gaming platforms to provide insights into market performance, monetization trends, and engine efficiency. By utilizing a highly normalized relational database structure, Nexus ensures data consistency across diverse entities. The system was tested using a cloud based MySQL environment, demonstrating high speed query execution and robust data integrity even with complex multi-platform listings.

Introduction
As the gaming industry continues its rapid expansion, the volume of data generated by multi-platform releases has become overwhelming. Information regarding development budgets, engine licenses, and player reviews is often scattered across different storefronts and internal records. This fragmentation makes it difficult for industry analysts to get a clear picture of a game's success or a studio's performance.

The Nexus Game Intelligence System addresses these challenges by creating a secure and unified core for gaming data. Beyond simple storage, the system implements advanced database features like atomic transactions and scalar functions to protect sensitive financial records and automate complex performance metrics. The goal is to provide a seamless and professional platform that turns raw data into actionable intelligence.

Methodology
Design (ER Diagram, Schema, Normalization)
ER Diagram: The architectural design centers on the Game entity, which connects to Developers, Publishers, and Game Engines. We used bridge entities like GameGenre and GamePlatformListing to handle Many to Many relationships, ensuring a clean and logical data flow.

Schema: We defined nine core tables with strict primary and foreign key constraints. This referential integrity ensures that every game is tied to a valid developer and engine, preventing orphaned data.

Normalization: The entire database was refined to meet the Third Normal Form (3NF). We eliminated all transitive dependencies and ensured that every non key attribute is fully dependent on the primary key, which minimizes data redundancy and prevents anomalies during updates.

Implementation
The system is built using a Node.js backend and a MySQL database hosted on Aiven Cloud. This cloud based approach ensures that the intelligence system remains accessible and scalable.

Algorithm for Game Registration (Atomic Transaction):
Input: Game Title, Developer ID, Publisher ID, Genre ID, and Price.
Output: Success confirmation and Unique Game ID.
Step 1: Open a secure SQL Transaction.
Step 2: Insert the primary game details into the Games table.
Step 3: Retrieve the newly generated Game ID using the LAST_INSERT_ID function.
Step 4: Map the game to its primary genre and create the initial platform listing.
Step 5: If all steps succeed, commit the changes; otherwise, trigger a rollback to maintain database safety.

Discussion/Results/Observations
The Nexus system was evaluated using realistic datasets featuring major industry titles. Performance benchmarks were conducted to compare our optimized indexing approach against a standard non indexed system.

Observations: The implementation of B-Tree indexing on the Title and Release Date columns significantly reduced latency as the dataset grew. The use of stored procedures also reduced network overhead by executing complex scoring logic directly on the server.

Concluding Remarks
The Nexus Game Intelligence System successfully demonstrates how a structured and optimized DBMS can bring clarity to the disorganized world of gaming data. Through automation and careful architectural design, we have created a tool that is both powerful and reliable.
